La version 3 del bloque de memoria compartida estara destinado a la automatizacion. Se unificara el bloque de momoria simple con el automatico.

-El nombre de la memoria no sera automatico. El usuario debera elegir el nombre de la memoria de entrada y de salida. 
Se debera asegurar que las memorias cargadas no esten en el contenedor.
El manejo interno de las memorias sera transparente para el usuario (el numero de memoria que utilice no estara relacionado con el contenedor)
La elaboracion automatica del nombre se realizara en la mascara y no en la funcion. 

-Para la entrada , solo debera cargarse el numero de entradas.La dimension y el tipo se determinarian automaticamente. 

-Para las salidas se determinaran en primera intancia por el contenido de memoria, y en segunda instancia por el seteo del programa.  

-cabecera? la memoria podria contener una cabecera de tamaço fijo, que incluya cantidad de entradas, salidas , tipos y programas que han accedido a la memoria.
